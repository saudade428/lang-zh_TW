:diagnostics_1_plug:
此實用程序用於故障排除。它將收集所有系統信息和配置文件，並將這些文件打包到一個可以保存在本地的 ZIP 文件。
隨後，此文件可以包含在您與 Limetech 或 UnRAID 論壇的通信中。

這有助於其他人快速獲取您系統的內部信息，並為您的問題提供更好的支持。將收集以下信息
和配置文件:
:end

:diagnostics_2_plug:
+ */config*
<br>覆制所有的 *\*.cfg* 文件、*go* 文件和 *super.dat* 文件。這些都是是配置文件。
+ */config/shares*
<br>覆制所有的 *\*.cfg* 文件。這些是用戶共享設置文件。
+ *Syslog file(s)*
<br>覆制當前的 *syslog* 文件和以前存在的任何 *syslog* 文件。
+ *System*
<br>保存以下命令的輸出：
<br>lsscsi、lspci、lsusb、free、lsof、ps、ethtool 和 ifconfig。
<br>顯示 IOMMU 組。
<br>顯示命令行參數（例如 PCIe ACS 覆蓋、PCI 存根等）。
<br>保存系統變量。
+ *SMART reports*
<br>保存系統中存在的每個磁盤的 SMART 報告。
+ *Docker*
<br>保存文件 *docker.log*、*libvirtd.log*、*libvirt/qemu/\*.log* 和 *vfio-pci*。
:end

:diagnostics_3_plug:
單擊**下載**將啟動采集過程，然後指示您的瀏覽器將 zip 文件保存到本地。

*不包括任何個人信息，如用戶名、密碼或上面未指定的任何其他文件內容
通過 UnRAID OS；但是，您的服務器名稱、IP 地址和用戶共享名稱**將**包括在內。*

*請注意第三方插件**可能**也許不會將個人信息存儲在插件特定的配置文件和/或
系統日志輸出中。*

在公共論壇上發布診斷文件時*使用* **匿名診斷**。*在與 Limetech 的私人通信中，建議取消選中此設置，並捕獲所有未更改的信息。*
:end

:newconfig_plug:
該實用程序用於重置陣列磁盤配置，以使所有磁盤都顯示為“新”磁盤，
就好像它是全新的服務器一樣。

當您添加或刪除了多個驅動器並且希望基於新配置重建奇偶校驗時，
此功能很有用。

重置陣列***之後***，使用 *'保留當前分配'* 選項填充所需的磁盤插槽。默認情況下，**不**填充磁盤插槽。

**請勿使用此實用程序，因為它會重建故障驅動器** —— 它將產生相反的效果，
將***無法***重建現有的故障驅動器 —— 您已受到警告！
:end

:newperms_1_plug:
從 5.0 之前的 Unraid 服務器版本升級到 Unraid v5 或更高版本之後，必須執行這項一次性操作。
執行這項一次性操作時，選擇**所有磁盤**。

從 Active Directory 過渡到非 Active Directory 集成時，
該實用程序還可用於恢覆文件和目錄的默認所有權/權限。

該實用程序將啟動一個後台進程，該進程將訪問每個數據磁盤和緩存磁盤，
選擇用戶共享並將文件和目錄所有權更改為*無人/用戶*（即 uid / gid 更改為 99/100），
並按以下方式設置權限：
:end

:newperms_2_plug:
對於目錄：
drwxrwxrwx

對於讀/寫文件：
-rw-rw-rw-

對於只讀文件：
-r--r--r--
:end

:newperms_3_plug:
單擊**開始**將打開另一個窗口並啟動後台進程。

在完成之前關閉窗口將終止後台進程 - 因此請不要這樣做。如果您有許多文件，則此進程可能耗時較長。

請注意，如果允許包含您的應用數據共享，則此工具可能會對 Docker 容器產生不利影響。
:end
